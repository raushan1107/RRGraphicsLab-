💡 The Assembly Line Analogy

Think of a car factory. Steel enters one end, a finished car exits the other. Each station has exactly one job — weld, paint, fit windows. You don't redesign the conveyor belt; you only design the car parts. The rendering pipeline is identical: raw numbers (vertex positions) enter, coloured pixels exit. You only write code for two stations — Vertex Shader and Fragment Shader. Everything else is automatic GPU hardware.

The 7-Stage Rendering Pipeline
💡 Key Insight — Why Two Shader Stages?

Vertex Shader runs once per vertex (corner point). If you have a triangle with 3 corners, it runs 3 times. Its job: position. Where does this point sit on screen?

Fragment Shader runs once per pixel the triangle covers. A triangle covering 5,000 pixels runs 5,000 times — simultaneously on thousands of GPU cores. Its job: colour. What colour is this pixel?

This separation is why GPUs are so fast — the fragment shader is massively parallel, running on thousands of shader cores at the same time.

CMakeLists.txt — Line by Line
💡 What CMake Actually Does

CMake is not a compiler. It's a project generator. You write one CMakeLists.txt file, CMake reads it, and generates a Visual Studio .sln + .vcxproj file — the same files VS creates when you click "New Project". The advantage: one CMakeLists.txt works on Windows, Linux, and Mac. No manual project settings.

CMakeLists.txt
CMake
# Minimum CMake version required — older versions lack features we need
cmake_minimum_required(VERSION 3.20)

# Project name — becomes the .sln name and the .exe name
project(M02_Window)

# Compile all .cpp files as C++17
# C++17 adds: structured bindings, std::filesystem, if-constexpr
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# Find OpenGL — it's installed by your GPU driver, not by you
# REQUIRED = fail with clear error if not found
find_package(OpenGL REQUIRED)

# Read Windows Environment Variables we set during setup
# $ENV{} is CMake syntax for "read this environment variable"
# This avoids hardcoding paths like C:\libs\glfw on every machine
set(GLFW_DIR $ENV{GLFW_DIR})   # → C:\libs\glfw
set(GLEW_DIR $ENV{GLEW_DIR})   # → C:\libs\glew
set(GLM_DIR  $ENV{GLM_DIR})    # → C:\libs\glm

# Tell the compiler WHERE to find header (.h/.hpp) files
# Without this, #include <GLFW/glfw3.h> would fail — "file not found"
include_directories(
    ${GLFW_DIR}/include    # → C:\libs\glfw\include\GLFW\glfw3.h
    ${GLEW_DIR}/include    # → C:\libs\glew\include\GL\glew.h
    ${GLM_DIR}             # → C:\libs\glm\glm\glm.hpp
)

# Define our executable: name = M02_Window, source = src/main.cpp
add_executable(M02_Window src/main.cpp)

# Link = attach library binaries (.lib files) to our program
# Without linking: "unresolved external symbol glfwInit" linker error
# OpenGL::GL = opengl32.lib (found by find_package above)
# glfw3.lib  = pre-compiled GLFW for Visual Studio 2022
# glew32s.lib = GLEW static build ('s' suffix = no runtime .dll needed)
target_link_libraries(M02_Window
    OpenGL::GL
    ${GLFW_DIR}/lib-vc2022/glfw3.lib
    ${GLEW_DIR}/lib/Release/x64/glew32s.lib
)
main.cpp — Every Line Explained
src/main.cpp
C++
// GLEW must come BEFORE GLFW — it overrides some GL function definitions
// GLEW_STATIC = link the static library (.lib) — no .dll file needed at runtime
#define GLEW_STATIC
#include <GL/glew.h>
#include <GLFW/glfw3.h>
#include <iostream>

// ── CALLBACK: runs whenever window is resized ────────────────────────────────
// GLFW calls this automatically — you register it, GLFW calls it
// Parameters: the window handle, new pixel width, new pixel height
// glViewport maps GL's -1 to +1 space onto these pixel dimensions
void onResize(GLFWwindow* w, int width, int height) {
    glViewport(0, 0, width, height);
    // (0,0) = bottom-left corner of drawable area
    // width, height = full window size in pixels
}

// ── INPUT: check keyboard every frame ────────────────────────────────────────
// Called inside render loop = checked 60 times per second
void processInput(GLFWwindow* w) {
    // glfwGetKey returns GLFW_PRESS if key is currently held down
    // glfwSetWindowShouldClose(win, true) sets internal "please quit" flag
    // The render loop checks this flag via glfwWindowShouldClose()
    if (glfwGetKey(w, GLFW_KEY_ESCAPE) == GLFW_PRESS)
        glfwSetWindowShouldClose(w, true);
}

int main() {

    // ① INIT GLFW ─────────────────────────────────────────────────────────────
    // Must be called before any other GLFW function
    // Initialises GLFW's internal state, detects the OS/window system
    if (!glfwInit()) {
        std::cerr << "GLFW init failed\n";
        return -1;
    }
    std::cout << "[1/5] GLFW initialised\n";

    // ② CONFIGURE CONTEXT HINTS ──────────────────────────────────────────────
    // These "hints" tell GLFW what kind of OpenGL context to create
    // They must be set BEFORE glfwCreateWindow — they're like order specs
    glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 3); // OpenGL major version = 3
    glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 3); // OpenGL minor version = 3 → 3.3
    glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_CORE_PROFILE);
    // CORE_PROFILE = modern OpenGL only (no legacy glBegin/glEnd calls)
    // COMPAT_PROFILE = all legacy functions still available (not what we want)

    // ③ CREATE WINDOW + CONTEXT ──────────────────────────────────────────────
    // glfwCreateWindow creates BOTH:
    //   - A native OS window (via Win32 on Windows)
    //   - An OpenGL context attached to that window
    // Parameters: width, height, title, monitor(NULL=windowed), shareContext(NULL)
    GLFWwindow* window = glfwCreateWindow(800, 600,
        "RR Graphics Lab — Module 2", NULL, NULL);
    if (!window) {
        std::cerr << "Window creation failed\n";
        glfwTerminate();
        return -1;
    }
    std::cout << "[2/5] Window + GL context created\n";

    // ④ MAKE CONTEXT CURRENT ─────────────────────────────────────────────────
    // "Current" = all gl* calls on this thread now go to THIS context
    // CRITICAL: GLEW init will fail silently without a current context
    glfwMakeContextCurrent(window);
    glfwSetFramebufferSizeCallback(window, onResize);
    std::cout << "[3/5] Context made current, resize callback registered\n";

    // ⑤ INITIALISE GLEW ──────────────────────────────────────────────────────
    // GLEW goes into the GPU driver and finds function pointers for every
    // OpenGL function (glGenBuffers, glDrawArrays etc.)
    // Without this: calling any gl* function crashes the program
    glewExperimental = GL_TRUE; // needed to load all extensions reliably
    if (glewInit() != GLEW_OK) {
        std::cerr << "GLEW init failed\n";
        return -1;
    }
    std::cout << "[4/5] GLEW initialised — all OpenGL functions loaded\n";

    // Print hardware info — visible proof the context is live
    std::cout << "\n=== Hardware Info ===\n";
    std::cout << "GPU:     "            << glGetString(GL_RENDERER) << "\n";
    std::cout << "OpenGL:  "            << glGetString(GL_VERSION)  << "\n";
    std::cout << "GLSL:    "            << glGetString(GL_SHADING_LANGUAGE_VERSION) << "\n";
    std::cout << "Vendor:  "            << glGetString(GL_VENDOR)   << "\n";
    std::cout << "=====================\n\n";
    std::cout << "[5/5] Render loop starting — press ESC to quit\n";

    // ⑥ RENDER LOOP ──────────────────────────────────────────────────────────
    // Runs every frame — typically 60 times per second
    // Each iteration = one frame on screen
    while (!glfwWindowShouldClose(window)) {

        processInput(window);    // ① check input (ESC?)

        glClearColor(0.05f, 0.08f, 0.15f, 1.0f); // ② set clear colour (navy blue)
        glClear(GL_COLOR_BUFFER_BIT);               // ③ clear the back buffer

        // ④ draw calls go here (we add these from Demo 3 onward)

        glfwSwapBuffers(window);  // ⑤ show completed frame (back→front)
        glfwPollEvents();         // ⑥ process queued input events
    }

    glfwTerminate();
    return 0;
}

Part 1 of 4 — Shader Source Strings (GLSL)
💡 Shaders Live in Your C++ File as Strings

GLSL (OpenGL Shading Language) looks like C but runs on the GPU. You write it as a raw string inside your C++ file. OpenGL sends this string to the GPU driver which compiles it at runtime — not during cmake build. R"(...)" is C++11 raw string literal syntax that preserves newlines and quotes exactly.

src/main.cpp — Section 1: Shader Sources (add above main)
C++ + GLSL
#define GLEW_STATIC
#include <GL/glew.h>
#include <GLFW/glfw3.h>
#include <iostream>

// ─── VERTEX SHADER ──────────────────────────────────────────────────────────
// Runs ONCE PER VERTEX on the GPU.
// For a 3-vertex triangle: runs exactly 3 times simultaneously.
//
// #version 330 core → GLSL version 3.30, core profile (matches OpenGL 3.3)
// layout(location=0) → attribute index 0 — MUST match glVertexAttribPointer(0,...)
// in vec3 aPos → "in" = input from VBO. "aPos" = your name, can be anything.
// gl_Position → BUILT-IN required output. vec4 because GPU uses 4D math internally.
// vec4(aPos, 1.0) → promote vec3(x,y,z) to vec4(x,y,z,w) where w=1 for positions
const char* vertSrc = R"GLSL(
    #version 330 core
    layout(location = 0) in vec3 aPos;

    void main() {
        gl_Position = vec4(aPos, 1.0);
    }
)GLSL";

// ─── FRAGMENT SHADER ─────────────────────────────────────────────────────────
// Runs ONCE PER PIXEL (fragment) the triangle covers.
// For a triangle covering 5,000 pixels: runs 5,000 times simultaneously.
//
// out vec4 FragColor → your output. Name is your choice. Type must be vec4 (RGBA).
// vec4(1.0, 0.5, 0.2, 1.0) → orange: R=full, G=half, B=low, A=opaque
const char* fragSrc = R"GLSL(
    #version 330 core
    out vec4 FragColor;

    void main() {
        FragColor = vec4(1.0, 0.5, 0.2, 1.0);
    }
)GLSL";
Part 2 of 4 — Shader Compiler Helper + VBO/VAO Setup
src/main.cpp — Section 2: Helpers + Callbacks
C++
// ── Compile one shader, return its ID ────────────────────────────────────────
// type = GL_VERTEX_SHADER or GL_FRAGMENT_SHADER
// Always check compile status — GPU driver compiles GLSL at runtime
unsigned int compileShader(unsigned int type, const char* src) {
    unsigned int id = glCreateShader(type);
    glShaderSource(id, 1, &src, NULL);
    glCompileShader(id);
    int ok; char log[512];
    glGetShaderiv(id, GL_COMPILE_STATUS, &ok);
    if (!ok) {
        glGetShaderInfoLog(id, 512, NULL, log);
        std::cerr << "Shader error: " << log << "\n";
    }
    return id;
}

// ── Link vertex + fragment into a shader program, return program ID ───────────
unsigned int makeProgram(const char* vSrc, const char* fSrc) {
    unsigned int vs = compileShader(GL_VERTEX_SHADER,   vSrc);
    unsigned int fs = compileShader(GL_FRAGMENT_SHADER, fSrc);
    unsigned int prog = glCreateProgram();
    glAttachShader(prog, vs);
    glAttachShader(prog, fs);
    glLinkProgram(prog);
    glDeleteShader(vs); glDeleteShader(fs); // no longer needed after link
    return prog;
}

void onResize(GLFWwindow* w, int W, int H) { glViewport(0,0,W,H); }
void processInput(GLFWwindow* w) {
    if(glfwGetKey(w,GLFW_KEY_ESCAPE)==GLFW_PRESS) glfwSetWindowShouldClose(w,true);
}
Part 3 of 4 — main(): Window + VBO + VAO + Shaders
⚠️ Read Before You Type

Notice the ORDER: VAO bind → then VBO creation → then glVertexAttribPointer. The VAO must be bound FIRST because it records everything that follows. Think of it as pressing "record" before describing the layout. If you bind the VAO after the VBO, it won't know about it.

src/main.cpp — Section 3: main() — setup + VBO + VAO
C++
int main() {
    // ── Window setup (same as Demo 2) ─────────────────────────────────────────
    glfwInit();
    glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 3);
    glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 3);
    glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_CORE_PROFILE);
    GLFWwindow* win = glfwCreateWindow(800, 600, "Demo 3 — First Triangle", NULL, NULL);
    glfwMakeContextCurrent(win);
    glfwSetFramebufferSizeCallback(win, onResize);
    glewExperimental = GL_TRUE;
    glewInit();

    // ── Compile + link shader program ─────────────────────────────────────────
    unsigned int shader = makeProgram(vertSrc, fragSrc);
    std::cout << "[1] Shader program compiled + linked. ID = " << shader << "\n";

    // ── Vertex data: 3 positions in NDC ──────────────────────────────────────
    // NDC = Normalized Device Coordinates: -1.0 to +1.0 on both axes
    // (0,0) = screen centre. (+1,+1) = top-right. (-1,-1) = bottom-left.
    float verts[] = {
        // X       Y       Z
        -0.5f, -0.5f,  0.0f,   // bottom-left corner
         0.5f, -0.5f,  0.0f,   // bottom-right corner
         0.0f,  0.5f,  0.0f    // top-centre
    }; // 3 vertices × 3 floats = 9 floats = 36 bytes total

    // ── VAO — create and bind FIRST (recording begins) ───────────────────────
    // VAO = Vertex Array Object. Stores the complete description of:
    //   which VBO to use, how many floats per vertex, their stride/offset.
    // Binding it later (in render loop) restores all this state instantly.
    unsigned int VAO, VBO;
    glGenVertexArrays(1, &VAO);
    glBindVertexArray(VAO);      // ← RECORDING STARTS
    std::cout << "[2] VAO created and bound. ID = " << VAO << "\n";

    // ── VBO — create, bind, upload data (happens INSIDE VAO recording) ───────
    // Step 1: Generate — OpenGL allocates a buffer object on the GPU, returns ID
    glGenBuffers(1, &VBO);
    std::cout << "[3] VBO generated on GPU. ID = " << VBO << "\n";

    // Step 2: Bind — select this VBO as the active GL_ARRAY_BUFFER
    // OpenGL state machine: you select, then operate
    glBindBuffer(GL_ARRAY_BUFFER, VBO);

    // Step 3: Upload — copy 36 bytes from CPU RAM to GPU VRAM
    // After this call: GPU has a permanent copy. Your verts[] is no longer needed.
    glBufferData(GL_ARRAY_BUFFER, sizeof(verts), verts, GL_STATIC_DRAW);
    std::cout << "[4] " << sizeof(verts) << " bytes uploaded to GPU VRAM via VBO\n";

    // ── Tell GPU how to interpret those bytes ─────────────────────────────────
    // Right now the GPU has 36 raw bytes. It doesn't know if they're
    // positions, colours, or random data. glVertexAttribPointer explains:
    //
    //  Param 1 → attribute index 0 (must match layout(location=0) in vertex shader)
    //  Param 2 → 3 components per attribute (x, y, z)
    //  Param 3 → each component is a float (4 bytes)
    //  Param 4 → GL_FALSE: don't normalise — use raw float values
    //  Param 5 → stride: 12 bytes from one vertex's x to next vertex's x
    //               (3 floats × 4 bytes = 12)
    //  Param 6 → offset: position data starts at byte 0 of each vertex
    glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 3*sizeof(float), (void*)0);
    glEnableVertexAttribArray(0); // enable attr 0 — disabled by default
    std::cout << "[5] Vertex layout described to GPU (VAO recording complete)\n";

    glBindVertexArray(0);         // ← RECORDING STOPS — unbind VAO safely
    std::cout << "[6] Setup complete. Starting render loop...\n\n";
Part 4 of 4 — Render Loop (Draw the Triangle)
src/main.cpp — Section 4: Render Loop + Cleanup
C++
    while (!glfwWindowShouldClose(win)) {
        processInput(win);

        glClearColor(0.08f, 0.10f, 0.16f, 1.0f);
        glClear(GL_COLOR_BUFFER_BIT);

        // ① Activate our shader program — all draw calls now use these shaders
        // glUseProgram tells the GPU: "use this vertex + fragment shader pair"
        glUseProgram(shader);

        // ② Bind VAO — restores EVERYTHING in one call:
        //   - which VBO is bound
        //   - how to interpret its bytes (stride, offset, type)
        //   - which attributes are enabled
        // This is why VAO exists — one call replaces 4-5 setup calls
        glBindVertexArray(VAO);

        // ③ Draw — send vertices through the pipeline
        // GL_TRIANGLES  = every 3 vertices = 1 filled triangle
        // 0             = start at vertex index 0 in the VBO
        // 3             = process 3 vertices total → 1 triangle
        glDrawArrays(GL_TRIANGLES, 0, 3);

        glfwSwapBuffers(win);
        glfwPollEvents();
    }

    // Always free GPU resources before exit
    glDeleteVertexArrays(1, &VAO);
    glDeleteBuffers(1, &VBO);
    glDeleteProgram(shader);
    glfwTerminate();
    return 0;
}

VAO with Two Attributes — Per-Vertex Colour

Interleaved VBO — Position + Colour per Vertex (6 floats = 24 bytes each)
  Byte: 0    4    8   12   16   20  │ 24   28   32   36   40   44  │ 48 ...
        [X0] [Y0] [Z0] [R0] [G0] [B0] │ [X1] [Y1] [Z1] [R1] [G1] [B1] │ ...
        ├───position──┤├──colour──┤   │ ├───position──┤├──colour──┤   │
               Vertex 0 (24 bytes)    │          Vertex 1 (24 bytes)   │

  Stride for BOTH attributes = 24 bytes (6 floats × 4 bytes)

  Attribute 0 (position): offset = 0  bytes — starts at beginning of each vertex
  Attribute 1 (colour):   offset = 12 bytes — starts after the 3 position floats
src/main.cpp — Updated shaders + vertex data + VAO setup
C++
// ── UPDATED VERTEX SHADER — now receives two attributes ──────────────────────
// location=0 → position (same as before)
// location=1 → colour   (NEW — matches glVertexAttribPointer(1,...))
// "out vec3 vColour" → pass colour to fragment shader (interpolated automatically)
const char* vertSrc = R"GLSL(
    #version 330 core
    layout(location = 0) in vec3 aPos;
    layout(location = 1) in vec3 aColour;

    out vec3 vColour;       // passed to fragment shader, GPU interpolates between vertices

    void main() {
        gl_Position = vec4(aPos, 1.0);
        vColour = aColour;  // just pass it through — no change
    }
)GLSL";

// ── UPDATED FRAGMENT SHADER — receives interpolated colour ────────────────────
// "in vec3 vColour" matches "out vec3 vColour" in vertex shader — same name required
const char* fragSrc = R"GLSL(
    #version 330 core
    in vec3 vColour;
    out vec4 FragColor;

    void main() {
        FragColor = vec4(vColour, 1.0);  // RGB from vertex, Alpha=1 (opaque)
    }
)GLSL";

// ── INTERLEAVED VERTEX DATA: X Y Z R G B per vertex ─────────────────────────
float verts[] = {
    // X       Y       Z      R     G     B

Uniforms — Sending Live Data to Shaders Every Frame
A uniform is a value you send from C++ to the GPU every frame. Unlike vertex attributes (uploaded once), uniforms change each frame. Pass time → shader animates colour with sin(). Pass a position offset → triangle moves. This is how all real-time rendering works.
EXTENDS: Demo 3 triangle + Add uTime uniform to shader + Send glfwGetTime() each frame = Pulsing animated triangle ✓
🎯 What You Will See
Screen: Triangle smoothly cycling through colours continuously with no input needed
Terminal: Prints current time value every 60 frames to confirm uniform is being sent
src/main.cpp — Uniform animation (changes from Demo 3)
C++
// Updated fragment shader — receives uTime uniform from C++
// sin() returns -1 to +1 → (sin()+1)/2 maps to 0.0-1.0 for colour channels
const char* fragSrc = R"GLSL(
    #version 330 core
    out vec4 FragColor;

    uniform float uTime;    // sent from C++ every frame via glUniform1f

    void main() {
        // Each channel uses sin at different speed → colour cycling effect
        float r = (sin(uTime * 1.0) + 1.0) / 2.0;
        float g = (sin(uTime * 0.7 + 1.0) + 1.0) / 2.0;
        float b = (sin(uTime * 0.4 + 2.0) + 1.0) / 2.0;
        FragColor = vec4(r, g, b, 1.0);
    }
)GLSL";

// ── Before the render loop: get uniform location ──────────────────────────────
// glGetUniformLocation finds "uTime" by name in the compiled shader
// Call this ONCE before the loop — it's slow. Cache the result.
// Returns -1 if "uTime" doesn't exist in the shader (typo check)
int uTimeLoc = glGetUniformLocation(shader, "uTime");
std::cout << "uTime uniform location: " << uTimeLoc << " (−1 = not found)\n";

int frameCount = 0;

// ── Render loop ───────────────────────────────────────────────────────────────
while (!glfwWindowShouldClose(win)) {
    processInput(win);
    glClearColor(0.08f, 0.10f, 0.16f, 1.0f);
    glClear(GL_COLOR_BUFFER_BIT);
    glUseProgram(shader);

    // Send current time to GPU shader every frame
    // glfwGetTime() returns seconds since glfwInit() as a double
    // glUniform1f sets ONE float uniform at location uTimeLoc
    float t = (float)glfwGetTime();
    glUniform1f(uTimeLoc, t);

    // Print time every 60 frames — confirm it's updating
    if (++frameCount % 60 == 0)
        std::cout << "Frame " << frameCount << ": uTime = " << t << "s\n";

    glBindVertexArray(VAO);
    glDrawArrays(GL_TRIANGLES, 0, 3);
    glfwSwapBuffers(win);
    glfwPollEvents();
}

#define GLEW_STATIC
#include <GL/glew.h>
#include <GLFW/glfw3.h>
#include <iostream>
#include <cmath>   // for sin()

// ── VERTEX SHADER ─────────────────────────────────────────────────────────────
// Attributes:  location=0 → position (vec3), location=1 → colour (vec3)
// Uniforms:    uPulse → brightness multiplier sent from CPU each frame
// Out:         vColour → interpolated colour passed to fragment shader
const char* vertSrc = R"GLSL(
    #version 330 core
    layout(location = 0) in vec3 aPos;
    layout(location = 1) in vec3 aColour;

    out vec3 vColour;

    uniform float uPulse;   // 0.0 to 1.0 — brightness multiplier from CPU

    void main() {
        gl_Position = vec4(aPos, 1.0);
        vColour = aColour * uPulse;   // scale colour brightness with pulse
    }
)GLSL";

// ── FRAGMENT SHADER ─────────────────────────────────────────────────────────
// Simple pass-through — colour already computed in vertex shader
const char* fragSrc = R"GLSL(
    #version 330 core
    in vec3 vColour;
    out vec4 FragColor;

    void main() {
        FragColor = vec4(vColour, 1.0);
    }
)GLSL";

// ── HELPERS ───────────────────────────────────────────────────────────────────
unsigned int compileShader(unsigned int type, const char* src) {
    unsigned int id = glCreateShader(type);
    glShaderSource(id, 1, &src, NULL); glCompileShader(id);
    int ok; char log[512];
    glGetShaderiv(id, GL_COMPILE_STATUS, &ok);
    if (!ok) { glGetShaderInfoLog(id,512,NULL,log); std::cerr << "Shader:\n"<<log; }
    return id;
}
unsigned int makeProgram(const char* v, const char* f) {
    unsigned int vs=compileShader(GL_VERTEX_SHADER,v), fs=compileShader(GL_FRAGMENT_SHADER,f);
    unsigned int p=glCreateProgram();
    glAttachShader(p,vs); glAttachShader(p,fs); glLinkProgram(p);
    glDeleteShader(vs); glDeleteShader(fs);
    return p;
}
void onResize(GLFWwindow* w,int W,int H){glViewport(0,0,W,H);}
void processInput(GLFWwindow* w){
    if(glfwGetKey(w,GLFW_KEY_ESCAPE)==GLFW_PRESS)glfwSetWindowShouldClose(w,true);
}

    bool wire = false;
    bool wPrev = false;

    while (!glfwWindowShouldClose(win)) {
        processInput(win);

        // W key: toggle wireframe (edge-detect keypress with prev state)
        bool wNow = (glfwGetKey(win, GLFW_KEY_W) == GLFW_PRESS);
        if (wNow && !wPrev) {
            wire = !wire;
            glPolygonMode(GL_FRONT_AND_BACK, wire ? GL_LINE : GL_FILL);
            std::cout << (wire ? "Wireframe ON\n" : "Wireframe OFF\n");
        }
        wPrev = wNow;

        glClearColor(0.06f, 0.08f, 0.14f, 1.0f);
        glClear(GL_COLOR_BUFFER_BIT);

        // ── Send pulse uniform ─────────────────────────────────────────────────
        // sin() oscillates -1 to +1 → (sin()+1)/2 maps to 0.0 - 1.0
        // This drives the brightness of both shapes in sync
        float t     = (float)glfwGetTime();
        float pulse = (std::sin(t * 2.0f) + 1.0f) / 2.0f; // 0.0 to 1.0
        pulse = 0.4f + pulse * 0.6f; // clamp to 0.4-1.0 so shapes never go fully black

        glUseProgram(shader);
        glUniform1f(uPulseLoc, pulse);  // send to vertex shader every frame
