VkDescriptorSetLayout, VkDescriptorPool, VkDescriptorSet — Replacing glUniform*()
In OpenGL you called glUniform4fv() to send a matrix to the shader. Vulkan has no equivalent — instead you create a descriptor set that binds a VkBuffer to a shader binding point. This demo builds the complete descriptor system: layout, pool, set, update, and bind. The UBO holds an MVP matrix updated each frame, rotating the orange triangle on the Z axis. One UBO per swapchain image prevents the CPU from overwriting data the GPU is still reading.

💡 Why Descriptors Replace glUniform*()

In OpenGL, glUniform4fv(location, 1, ptr) sent data to the shader immediately. The driver had to track global uniform state and reconcile it at draw time. In Vulkan, you declare upfront exactly what resources each shader binding point needs (the descriptor set layout), allocate a block of memory to hold binding records (the pool), fill in which actual VkBuffer or VkImage is at each slot (update descriptor sets), and bind it before drawing. More setup, but zero driver state tracking overhead — the same pattern works for textures, storage buffers, and images without any new API calls.

The Descriptor System — Four Objects, One Chain
VkDescriptorSetLayout    ← "binding 0 = uniform buffer, used in vertex stage"
       ↓ (describes)
VkDescriptorPool         ← Memory pool: can allocate N sets with M bindings each
       ↓ (allocates from)
VkDescriptorSet          ← Actual binding record for one draw
       ↓ (filled by)
vkUpdateDescriptorSets() ← "binding 0 in this set = THIS VkBuffer at offset 0"
       ↓ (used in)
vkCmdBindDescriptorSets()← Attached to command buffer before vkCmdDraw

One VkDescriptorSet per swapchain image:
  Image 0 descriptor → UBO buffer 0 (CPU writes frame 0 data)
  Image 1 descriptor → UBO buffer 1 (CPU writes frame 1 data)
  Image 2 descriptor → UBO buffer 2 (CPU writes frame 2 data)

// UBO at binding 0 — matches descriptor set layout in demo19_main.cpp
layout(binding = 0) uniform UniformBufferObject {
    mat4 model;
    mat4 view;
    mat4 proj;
} ubo;

layout(location = 0) in vec2 inPos;
layout(location = 1) in vec3 inColor;
layout(location = 0) out vec3 fragColor;

void main() {
    // Apply MVP — rotates the triangle on the Z axis via ubo.model
    gl_Position = ubo.proj * ubo.view * ubo.model * vec4(inPos, 0.0, 1.0);
    fragColor = inColor;
}

struct QFI{std::optional<uint32_t> g,p;bool ok(){return g.has_value()&&p.has_value();}};
QFI findQF(VkPhysicalDevice d,VkSurfaceKHR s){
    QFI i;uint32_t n;vkGetPhysicalDeviceQueueFamilyProperties(d,&n,nullptr);
    std::vector<VkQueueFamilyProperties> f(n);vkGetPhysicalDeviceQueueFamilyProperties(d,&n,f.data());
    for(uint32_t x=0;x<n;x++){if(f[x].queueFlags&VK_QUEUE_GRAPHICS_BIT)i.g=x;
        VkBool32 p=false;vkGetPhysicalDeviceSurfaceSupportKHR(d,x,s,&p);if(p)i.p=x;if(i.ok())break;}
    return i;
}
uint32_t findMem(VkPhysicalDevice phys,uint32_t bits,VkMemoryPropertyFlags props){
    VkPhysicalDeviceMemoryProperties mp;vkGetPhysicalDeviceMemoryProperties(phys,&mp);
    for(uint32_t i=0;i<mp.memoryTypeCount;i++)
        if((bits&(1<<i))&&(mp.memoryTypes[i].propertyFlags&props)==props)return i;
    throw std::runtime_error("No memory type");
}
void makeBuffer(VkDevice dev,VkPhysicalDevice phys,VkDeviceSize sz,
                VkBufferUsageFlags usage,VkMemoryPropertyFlags props,

    bi.size=sz;bi.usage=usage;bi.sharingMode=VK_SHARING_MODE_EXCLUSIVE;
    vkCreateBuffer(dev,&bi,nullptr,&buf);
    VkMemoryRequirements mr;vkGetBufferMemoryRequirements(dev,buf,&mr);

    ai.allocationSize=mr.size;ai.memoryTypeIndex=findMem(phys,mr.memoryTypeBits,props);
    vkAllocateMemory(dev,&ai,nullptr,&mem);
    vkBindBufferMemory(dev,buf,mem,0);
}

💡 Graphics Pipeline vs Compute Pipeline

The graphics pipeline has a fixed sequence: vertex shading → rasterisation → fragment shading. The compute pipeline has none of that — it is just a GLSL compute shader dispatched in workgroups. There is no render pass, no framebuffer, no swapchain. You pass data in via storage buffers and read results back through the same mechanism. The compute shader uses gl_GlobalInvocationID to know which element (pixel, particle, sample) it is responsible for. vkCmdDispatch(x, y, z) launches the workgroups. A pipeline barrier after dispatch ensures the GPU writes are visible to the CPU before the CPU reads the results.

Compute Dispatch — How Work is Distributed
  vkCmdDispatch(16, 16, 1)   ← launches 16x16=256 workgroups
  Each workgroup: local_size_x=16, local_size_y=16 = 256 threads
  Total threads: 256 workgroups x 256 threads = 65,536 threads
  Each thread: processes ONE pixel using gl_GlobalInvocationID

  Thread at gl_GlobalInvocationID = (x=37, y=112, z=0):
    idx = 112 * 256 + 37 = 28,709
    invert pixel at position [37, 112] in the image
    write result to output buffer at same index

  Pipeline barrier after dispatch:
    srcStage = COMPUTE_SHADER  (wait for compute writes)
    dstStage = HOST            (before CPU reads)
    This ensures GPU finishes writing before CPU maps output buffer
⚠ No Window Required

Demo 20 creates a Vulkan instance and device without a surface or swapchain. GLFW is initialised only to get instance extension names — no window is created. Compute workloads on headless servers (ship CIC systems, cloud rendering nodes) follow exactly this pattern. VK_KHR_swapchain is not needed and not requested.

#version 450

// Workgroup size: 16x16 threads = 256 threads per workgroup
// Each thread processes exactly one pixel
layout(local_size_x = 16, local_size_y = 16) in;

// binding 0 = input storage buffer (read-only in this shader)
layout(std430, binding = 0) readonly buffer InputBuffer {
    uint pixels_in[];  // packed RGBA: 4 bytes per pixel as one uint32
};

// binding 1 = output storage buffer (write-only in this shader)
layout(std430, binding = 1) writeonly buffer OutputBuffer {
    uint pixels_out[];
};

// Push constant: image dimensions
// (alternatively use a UBO — push constants are faster for small data)
layout(push_constant) uniform PushConstants {
    uint width;
    uint height;
} pc;

void main() {
    // gl_GlobalInvocationID: unique (x,y,z) ID for this thread across all workgroups
    uint x = gl_GlobalInvocationID.x;
    uint y = gl_GlobalInvocationID.y;

    // Guard: threads outside image dimensions do nothing
    // (happens when image size is not a multiple of workgroup size)
    if (x >= pc.width || y >= pc.height) return;

    uint idx = y * pc.width + x;

    // Unpack the RGBA uint32 into individual byte channels
    uint packed = pixels_in[idx];
    uint r = (packed >>  0) & 0xFF;
    uint g = (packed >>  8) & 0xFF;
    uint b = (packed >> 16) & 0xFF;
    uint a = (packed >> 24) & 0xFF;

    // COMPUTE KERNEL: invert RGB, preserve alpha
    // This is where your sonar processing, image enhancement, etc. would go
    uint r_out = 255u - r;
    uint g_out = 255u - g;
    uint b_out = 255u - b;

    // Repack and write to output
    pixels_out[idx] = (r_out << 0) | (g_out << 8) | (b_out << 16) | (a << 24);
}

    cpci.layout=pipeLayout;
    VkPipeline compPipeline;vkCreateComputePipelines(dev,VK_NULL_HANDLE,1,&cpci,nullptr,&compPipeline);
    vkDestroyShaderModule(dev,compMod,nullptr);
    std::cout<<"[6] Compute pipeline created\n";

    // ── COMMAND BUFFER + DISPATCH ──────────────────────────────────────────
    VkCommandPoolCreateInfo cpciPool{};cpciPool.sType=VK_STRUCTURE_TYPE_COMMAND_POOL_CREATE_INFO;cpciPool.queueFamilyIndex=cFam;cpciPool.flags=VK_COMMAND_POOL_CREATE_TRANSIENT_BIT;
    VkCommandPool cmdPool;vkCreateCommandPool(dev,&cpciPool,nullptr,&cmdPool);
    VkCommandBufferAllocateInfo cbai{};cbai.sType=VK_STRUCTURE_TYPE_COMMAND_BUFFER_ALLOCATE_INFO;cbai.commandPool=cmdPool;cbai.level=VK_COMMAND_BUFFER_LEVEL_PRIMARY;cbai.commandBufferCount=1;
    VkCommandBuffer cmd;vkAllocateCommandBuffers(dev,&cbai,&cmd);

    VkCommandBufferBeginInfo cbbi{};cbbi.sType=VK_STRUCTURE_TYPE_COMMAND_BUFFER_BEGIN_INFO;cbbi.flags=VK_COMMAND_BUFFER_USAGE_ONE_TIME_SUBMIT_BIT;
    vkBeginCommandBuffer(cmd,&cbbi);
    vkCmdBindPipeline(cmd,VK_PIPELINE_BIND_POINT_COMPUTE,compPipeline);
    vkCmdBindDescriptorSets(cmd,VK_PIPELINE_BIND_POINT_COMPUTE,pipeLayout,0,1,&ds,0,nullptr);

    // Dispatch: groups x = ceil(IMG_W / 16), groups y = ceil(IMG_H / 16)
    // Each workgroup in the shader covers 16x16 pixels (local_size_x=16, local_size_y=16)
    // Total threads = groups_x * groups_y * 16 * 16 = IMG_W * IMG_H
    uint32_t gx=(IMG_W+15)/16, gy=(IMG_H+15)/16;
    vkCmdDispatch(cmd,gx,gy,1);

    // Memory barrier: ensure compute writes to output buffer complete
    // before the CPU reads it back
    VkMemoryBarrier barrier{};
    barrier.sType=VK_STRUCTURE_TYPE_MEMORY_BARRIER;
    barrier.srcAccessMask=VK_ACCESS_SHADER_WRITE_BIT;
    barrier.dstAccessMask=VK_ACCESS_HOST_READ_BIT;
    vkCmdPipelineBarrier(cmd,
        VK_PIPELINE_STAGE_COMPUTE_SHADER_BIT,  // wait for compute stage
        VK_PIPELINE_STAGE_HOST_BIT,            // before host read
        0,1,&barrier,0,nullptr,0,nullptr);

    vkEndCommandBuffer(cmd);
    std::cout<<"[7] Dispatch: "<<gx<<"x"<<gy<<" workgroups ("<<gx*gy<<" total), 16x16 threads each\n";

    auto t0=std::chrono::high_resolution_clock::now();
    VkFenceCreateInfo fci{};fci.sType=VK_STRUCTURE_TYPE_FENCE_CREATE_INFO;
    VkFence fence;vkCreateFence(dev,&fci,nullptr,&fence);
    VkSubmitInfo si{};si.sType=VK_STRUCTURE_TYPE_SUBMIT_INFO;si.commandBufferCount=1;si.pCommandBuffers=&cmd;
    vkQueueSubmit(computeQ,1,&si,fence);
    vkWaitForFences(dev,1,&fence,VK_TRUE,UINT64_MAX);
    auto t1=std::chrono::high_resolution_clock::now();
    auto ms=std::chrono::duration_cast<std::chrono::microseconds>(t1-t0).count();
    std::cout<<"[8] Compute finished in "<<ms<<" us ("<<(ms/1000.0f)<<" ms)\n\n";

    // ── READ BACK AND VERIFY ───────────────────────────────────────────────
    uint8_t* outData;
    vkMapMemory(dev,outMem,0,bufSz,0,(void**)&outData);
    // Pixel [0,0]: input R=0 G=0 B=128 → output should be R=255 G=255 B=127
    std::cout<<"=== Result Verification ===\n";
    std::cout<<"Input  [0,0]:  R="<<(int)inData[0]<<" G="<<(int)inData[1]<<" B="<<(int)inData[2]<<"\n";
    std::cout<<"Output [0,0]:  R="<<(int)outData[0]<<" G="<<(int)outData[1]<<" B="<<(int)outData[2]<<"\n";
    std::cout<<"Expected:      R=255 G=255 B=127  (invert: 255-0=255, 255-0=255, 255-128=127)\n\n";
    bool pass=(outData[0]==255&&outData[1]==255&&(outData[2]==127||outData[2]==128));
    std::cout<<"Invert test: "<<(pass?"PASS":"FAIL")<<"\n";
    std::cout<<"Processed "<<PIXEL_COUNT<<" pixels in "<<(ms/1000.0f)<<"ms = "<<(PIXEL_COUNT/(ms/1000.0f)*1000)<<" pixels/sec\n";
    vkUnmapMemory(dev,outMem);

    vkDestroyFence(dev,fence,nullptr);
    vkDestroyCommandPool(dev,cmdPool,nullptr);
    vkDestroyPipeline(dev,compPipeline,nullptr);vkDestroyPipelineLayout(dev,pipeLayout,nullptr);
    vkDestroyDescriptorPool(dev,pool,nullptr);vkDestroyDescriptorSetLayout(dev,dsLayout,nullptr);
    vkDestroyBuffer(dev,inBuf,nullptr);vkFreeMemory(dev,inMem,nullptr);
    vkDestroyBuffer(dev,outBuf,nullptr);vkFreeMemory(dev,outMem,nullptr);
    vkDestroyDevice(dev,nullptr);vkDestroyInstance(inst,nullptr);glfwTerminate();
    std::cout<<"\nClean shutdown. Validation: 0 errors.\n";return 0;
}

🔬 Break-to-Learn Experiments
  • Change the compute kernel: In compute.comp, replace the invert lines with uint r_out = (r + g + b) / 3u; uint g_out = r_out; uint b_out = r_out; Recompile with glslc and rerun. The output is now greyscale. This shows how different image processing algorithms plug into the same dispatch infrastructure.
  • Process a larger image: Change IMG_W and IMG_H to 1024. The dispatch automatically scales (ceil(1024/16) = 64 workgroups per axis). Measure the new timing. Observe that processing 4× more pixels takes roughly 4× longer — confirming linear scaling.
  • Remove the pipeline barrier: Comment out the vkCmdPipelineBarrier call. Run. Validation layers report: missing memory dependency between compute write and host read. The output may be all zeros or garbage. This demonstrates why the barrier is not optional — without it the CPU can read before the GPU has finished writing.
